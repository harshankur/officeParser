OfficeGenerator & RAG Specification

The OfficeGenerator transforms a universal OfficeParserAST into destination formats. It is format-agnostic and can also generate AI-ready chunking output for RAG pipelines.

OfficeGenerator.generate()

Transforms an existing AST into a destination format. Returns a ConversionResult.

import { OfficeParser, OfficeGenerator } from 'officeparser';

const ast = await OfficeParser.parseOffice('report.docx');

// Convert to Markdown
const { value: md } = await OfficeGenerator.generate(ast, 'md');

// Convert to HTML with style mapping
const { value: html } = await OfficeGenerator.generate(ast, 'html', {
    includeFormatting: true,
    styleMap: [
        {
            selector: { nodeType: 'paragraph', attributes: { style: 'Heading 1' } },
            output: { tag: 'h1', classes: ['main-title'] }
        }
    ]
});

// Convert to CSV
const { value: csv } = await OfficeGenerator.generate(ast, 'csv');

Supported destinations: 'text' · 'md' · 'html' · 'csv' · 'rtf' · 'pdf' · 'chunks'

[!IMPORTANT] PDF generation requires the optional puppeteer peer dependency: npm install puppeteer

ast.to() — Shorthand

Every OfficeParserAST exposes a .to(destination, config?) method as a shorthand for OfficeGenerator.generate. This is the preferred way when you already have an AST:

const ast = await OfficeParser.parseOffice('report.docx');

const { value: markdown } = await ast.to('md');
const { value: html }     = await ast.to('html', { includeFormatting: false });
const { value: chunks }   = await ast.to('chunks', { strategy: 'fixed-size', chunkSize: 800 });
const { value: pdfBytes } = await ast.to('pdf'); // returns Uint8Array

ConversionResult Reference

All generation methods return this object, which includes the output and any diagnostic messages.

Property Type Description
value string | Uint8Array | OfficeChunk[] The actual generated content. Type depends on destination: pdf → Uint8Array, chunks → OfficeChunk[], csv → string | Uint8Array, all others → string.
messages OfficeIssue[] Array of warnings or info messages encountered during generation. For OfficeConverter.convert(), this also includes parsing-phase warnings.

RAG & Chunking Strategies

When the destination format is 'chunks', the generator returns an array of OfficeChunk objects. These are optimized for ingestion into Vector Databases for RAG pipelines.

The chunksConfig is a discriminated union — options differ per strategy.

1. Document Structure Chunking (Default)

Splits at natural AST boundaries (paragraphs, headings, pages, slides). Preserves logical flow. This is the recommended strategy for most use cases.

const { value: chunks } = await ast.to('chunks', {
    strategy: 'document-structure',
    splitBy: 'heading',    // 'paragraph' | 'heading' | 'page' | 'slide' | 'sheet'
    maxChunkSize: 1500,
    tableSplitStrategy: 'row' // repeats header row in every chunk — ideal for RAG
});

2. Fixed-Size Chunking

Splits text by character count with a sliding window overlap. Equivalent to LangChain's RecursiveCharacterTextSplitter.

const { value: chunks } = await ast.to('chunks', {
    strategy: 'fixed-size',
    chunkSize: 1000,
    chunkOverlap: 200,
    separators: ['\n\n', '\n', ' ', '']
});
console.log(`Generated ${chunks.length} chunks`);

3. Semantic Chunking

Uses embedding similarity to find natural topic breakpoints. Computationally expensive but produces the highest quality chunks. Requires an embeddingFunction.

import OpenAI from 'openai';
const openai = new OpenAI();

const { value: chunks } = await ast.to('chunks', {
    strategy: 'semantic',
    embeddingFunction: async (text) => {
        const res = await openai.embeddings.create({
            input: text, model: 'text-embedding-3-small'
        });
        return res.data[0].embedding;
    },
    similarityThreshold: 0.8,  // Default: 0.8 (higher = more splits)
    maxChunkSize: 2000,
    embeddingBatchSize: 50      // sentences per API batch
});

OfficeChunk Object

Every chunk contains the text content and rich metadata for citations and filtered retrieval:

Property Type Description
text string The chunk text content.
metadata.sourceType string The source file type (e.g., 'docx', 'pdf').
metadata.pageNumber number? Page number this chunk originates from (PDF).
metadata.slideNumber number? Slide number this chunk originates from (PPTX/ODP).
metadata.sheetName string? Sheet name this chunk originates from (XLSX/ODS).
metadata.closestHeading string? The nearest heading above this chunk. Invaluable for citations.
metadata.chunkIndex number Sequential index of this chunk within the document.
metadata.startIndex number? Character offset of the chunk start in the full document text (only when addStartIndex: true).

🚀 Performance Note

The OfficeGenerator is highly optimized. Once a document is parsed into an AST, generating multiple formats is extremely fast as it involves a single traversal of the tree. Parse once, generate many times with OfficeGenerator.generate(ast, ...) or ast.to(...).
