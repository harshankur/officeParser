OfficeGenerator & RAG Specification

The OfficeGenerator is the engine that transforms the universal OfficeParserAST into destination formats. It is designed to be format-agnostic and supports advanced AI-ready chunking strategies.

OfficeGenerator.generate()

Transforms an existing AST into a destination format.

import { OfficeGenerator } from 'officeparser';

const result = await OfficeGenerator.generate(ast, 'html', config);
console.log(result.value); // The generated string or buffer

RAG & Chunking Strategies

When the destination format is 'chunks', the generator returns an array of OfficeChunk objects. These are optimized for ingestion into Vector Databases for RAG pipelines.

1. Document Structure Chunking (Recommended)

Splits at natural boundaries (paragraphs/headings) while respecting a maximum size limit.

chunksConfig: {
    strategy: 'document-structure',
    splitBy: 'paragraph',
    maxChunkSize: 1000,
    tableSplitStrategy: 'row'
}

2. Fixed-Size Chunking

Splits text by character count with a sliding window overlap.

3. Semantic Chunking

Uses embedding similarity to find topic breakpoints. Requires an external embeddingFunction.

ConversionResult Reference

All generation methods return this object, which includes the output and any diagnostic messages.

Property Type Description
value string | Buffer | OfficeChunk[] The actual generated content.
messages OfficeIssue[] Array of warnings or info messages encountered during generation.

🚀 Performance Note

The OfficeGenerator is highly optimized. Once a document is parsed into an AST, generating multiple formats is extremely fast as it involves a single traversal of the tree.
