Generator Configuration Specification

The GeneratorConfig object controls how the AST is transformed into final output formats like HTML, Markdown, or PDF. It consists of common options and specialized sub-configurations.

Related: OfficeGenerator & ast.to() · OfficeParserConfig (the parse side) · AST structure · OfficeConverter · CLI flags

CommonGeneratorConfig

These settings apply to every generator unless specifically overridden in a sub-config.

Property Type Default Description
includeFormatting boolean true Visual Fidelity: Whether to include colors, font sizes, and text decorations (underline, strike, etc.) in the output.
generateIds boolean true Navigation: Automatically generates unique slug-based IDs for heading nodes, enabling internal anchor links.
renderMetadata boolean false Branding: If true, renders the document metadata (Title, Author, Creation Date) as a visible header block.
metadataOverrides MetadataOverrides {} Output Metadata: Overrides the metadata embedded in the generated file, merged per field over ast.metadata without mutating it. See MetadataOverrides for the full field list.
includeImages boolean true Resources: Whether to include <img> tags or Markdown image syntax in the output.
includeCharts boolean true Visuals: (HTML Only) Whether to render interactive charts. Requires Chart.js.
ignoreInternalLinks boolean false Cleanup: Suppresses all internal bookmarks and anchor references.
ignoreDefaultStyleMap boolean false Control: If true, ignores the built-in mapping of standard Word/Office styles to semantic tags.
styleMap string[] | object[] [] Semantic Bridge: Maps source styles (e.g., 'Intense Quote') to semantic tags (e.g., 'blockquote').
onNode function undefined Interception: Async callback called for every node during generation. Return false to skip, a string to override rendering, or void to proceed with default logic (mutations to node are applied).
onWarning function undefined Diagnostics: Callback for non-fatal issues encountered during generation (e.g. unsupported drawing, skipped node). Receives an OfficeIssue object.
abortSignal AbortSignal | null null Cancellation: An optional signal to abort the generation operation. Supported by PdfGenerator and ChunkingGenerator. Rejects with an AbortError.

Specialized Sub-Configurations

Format-specific settings nested under keys like htmlConfig, mdConfig, etc.

HtmlGeneratorConfig (htmlConfig)

Property Type Default Description
standalone boolean | StandaloneConfig true Controls the HTML "document envelope". true/omitted turns every part on (a full <html> shell with CSS/scripts/metadata); false turns every part off (a bare content fragment). Pass a StandaloneConfig object for granular control — see the table below.
chartJsSrc string (CDN) URL for the Chart.js script.
containerWidth string | number 'auto' Controls the maximum width of the rendered content container. Accepts a positive number (treated as pixels), a CSS length string ('900px', '100%', '60vw'), or 'auto' (browser default). Invalid values fall back to 'auto' and emit an INVALID_CONTAINER_WIDTH warning.
customCss string '' Raw CSS string injected into the generated <style> block. Use this to override built-in styles or add document-specific rules without modifying the core generator.
injections HtmlInjectionConfig {} Granular raw HTML injection points:
  • headStart: Injected immediately after <head> (requires standalone.document + standalone.headInjections)
  • headEnd: Injected immediately before </head> (same requirement)
  • bodyStart: Injected immediately after the opening body content (applies even to a bare fragment when standalone.bodyInjections is on)
  • bodyEnd: Injected immediately before the closing body content (same)
Useful for inserting analytics scripts, custom fonts, or wrapper elements.

StandaloneConfig (the object form of standalone)

Any field left unspecified defaults to its "on" (standalone) value, so { document: false } still yields a fully styled fragment — only the <html> shell is dropped.

Property Type Default Description
document boolean true Wraps output in <!DOCTYPE html><html><head>…</head><body>…</body></html>. When false, only the inner content fragment is emitted.
metaTags boolean true Emits <title> and <meta> tags. Only meaningful when document is true.
styles 'full' | 'scoped' | 'none' 'full' How the built-in stylesheet is delivered:
  • 'full' — the complete stylesheet using global selectors (body, h1, table, …) — what standalone: true has always emitted.
  • 'scoped' — the same styling, scoped under the fragment's own wrapper via CSS @scope so it cannot leak onto a host page's elements. Requires a modern engine (Chrome 118+, Safari 17.4+, Firefox 128+).
  • 'none' — no stylesheet; the host page/editor/EPUB reader supplies its own.
The boolean shorthand for standalone maps true → 'full', false → 'none'.
scripts boolean true Emits the Chart.js CDN loader and spreadsheet-interactivity <script> tags. (A per-chart inline init script is content, not envelope, and is emitted regardless of this flag whenever a chart node is present.)
headInjections boolean true Applies injections.headStart/headEnd. Only meaningful when document is true.
bodyInjections boolean true Applies injections.bodyStart/bodyEnd — applies even when generating a bare fragment, since these wrap body content, not the document shell.
Behavior change from previous versions
standalone: false previously emitted a fragment with a global, unscoped <style> block. It now emits a genuinely bare fragment (no <style> at all), matching "every part off". If you relied on the old styled-fragment behavior, pass { document: false } instead.

MdGeneratorConfig (mdConfig)

Property Type Default Description
dialect MarkdownDialectPreset | MarkdownDialectConfig 'extended' Which real-world Markdown flavor to emit. A preset name picks a whole set of behaviours at once; an object overrides individual features on top of one. See MarkdownDialectConfig below.
fallbackToHtml boolean | FallbackToHtmlConfig true Uses raw HTML tags for features Markdown cannot express (underline, alignment, merged cells). The boolean applies uniformly; an object controls each part independently. See FallbackToHtmlConfig below.

MarkdownDialectConfig

dialect accepts either a preset name or an object. The presets:

Preset Behaviour
'extended' (default)Every feature on, GitHub-style admonitions. Byte-identical to this library's output before dialects existed, so it is the backward-compatibility anchor.
'github'GFM: > [!NOTE] admonitions, footnotes, strikethrough, native tables. No definition lists, citations, wikilinks or attribute lists.
'gitlab'GLFM: :::note fenced-div admonitions, otherwise as GitHub.
'obsidian'As GitHub, plus [[wikilinks]].
'pandoc'::: {.note} admonitions, definition lists, citations and attribute lists on.
'commonmark'Strict CommonMark: no admonition syntax (plain blockquote), no footnote references (inlined as parentheticals), no strikethrough, and tables forced to HTML, since CommonMark has no table syntax.

Passing an object instead overrides individual features. Every field is optional; extends names the preset that any omitted field falls back to.

Property Type Description
extendsMarkdownDialectPresetBase preset for every field left unset. Defaults to 'extended'. Note this is the base for this object, not "whatever preset was configured before" — config merging replaces the whole dialect field rather than layering onto a prior string.
admonitions'github' | 'gitlab' | 'pandoc' | 'none'Which admonition syntax to emit. 'none' degrades to a bold-labelled blockquote.
math'dollar' | 'none'$inline$ / $$block$$ delimiters, or none (content emitted bare).
tables'native' | 'html'Pipe tables, or an HTML <table>. Forced to 'html' under commonmark.
bulletListMarker'-' | '*' | '+'Marker for unordered list items.
orderedListMarker'.' | ')'1. or 1) for ordered list items.
emphasisMarker'asterisk' | 'underscore'*em*/**strong** or _em_/__strong__.
definitionListsbooleanMarkdown Extra Term\n: Definition syntax. When off, definition lists degrade to plain paragraphs.
footnotesboolean[^id] references with a definition block. When off, note content is inlined as a parenthetical at the reference point rather than being dropped.
citationsbooleanPandoc [@citekey] syntax. When off, the key is emitted as plain [key].
wikilinksbooleanObsidian [[Page]] / [[Page|Alias]] syntax. When off, wikilinks become ordinary links.
attributeListsbooleanPandoc-style {width=50% align=center} after an image or table.
strikethroughboolean~~struck~~. When off, the formatting is dropped.
 
// A preset by name
await ast.to('md', { mdConfig: { dialect: 'github' } });

// GitHub, but with wikilinks and asterisk bullets
await ast.to('md', {
    mdConfig: { dialect: { extends: 'github', wikilinks: true, bulletListMarker: '*' } },
});

MetadataOverrides (metadataOverrides)

Overrides the metadata embedded in generated output, applied per field on top of ast.metadata (see the AST spec for the parsed shape), so setting one field leaves the rest of the parsed metadata intact. ast.metadata itself is never mutated, so the same AST can be generated repeatedly with different metadata.

Every field is optional. Field names mirror OfficeMetadata so the same vocabulary describes what was parsed and what gets written.

Property Type Written as
titlestringHTML <title>; EPUB dc:title; Markdown title:; RTF \title; the renderMetadata header in text/CSV
authorstringHTML <meta name="author">; EPUB dc:creator; Markdown author:; RTF \author
descriptionstringHTML <meta name="description">; EPUB dc:description; Markdown description:; RTF \comm
subjectstringHTML <meta name="DC.subject">; EPUB dc:subject; Markdown subject:; RTF \subject
keywordsstringHTML <meta name="keywords">; EPUB dc:subject; Markdown keywords:; RTF \keywords
lastModifiedBystringHTML <meta name="lastModifiedBy">
createdDateHTML <meta name="dcterms.created">; Markdown created:; the renderMetadata header in text/CSV
modifiedDateHTML <meta name="dcterms.modified">; EPUB dcterms:modified and every zip entry's mtime; Markdown modified:
languagestringEPUB dc:language. Stored on nativeProperties.language, where parsers already surface it.
customRecord<string, string | number | boolean | Date>Any caller-defined key/value pairs. HTML <meta name="custom:KEY">; Markdown frontmatter. Not representable in EPUB or RTF — see below.

custom is kept in its own bucket rather than mixed in beside the named fields on purpose: with a bare index signature a typo such as titel would silently become a custom entry instead of a compile error.

await ast.to('html', {
    metadataOverrides: {
        title: 'Q4 Report',
        author: 'Acme Inc',
        custom: { department: 'Finance', reviewed: true },
    },
});

Dates

created and modified take a Date. When modified is unset, officeParser uses the source document's own ast.metadata.modified, falling back to the current time only if the document has none. Dates outside the 1980–2099 range a ZIP timestamp can represent are clamped where EPUB writes them onto zip entries.

Not every format can represent every field

HTML <meta> tags and Markdown frontmatter are open vocabularies and accept anything. EPUB's OPF is a closed Dublin Core vocabulary and RTF's \info group has a fixed set of control words, so a custom entry has nowhere to go in either. Rather than dropping it silently, those generators report it through onWarning (OfficeWarningType.METADATA_NOT_REPRESENTABLE) and continue; the named fields still apply. Plain text and CSV only emit metadata at all when renderMetadata is on.

Style Mapping (The Semantic Bridge)

Style mapping allows you to define the semantic meaning of arbitrary document styles.

Legacy String DSL

["p[style-name='Heading 1'] => h1", "p[style='Quote'] => blockquote"]

Structured Objects (Recommended)

Use for full type safety, class application, and the ~= partial-match operator:

styleMap: [
    {
        selector: { nodeType: 'paragraph', attributes: { style: 'Heading 1' } },
        output: { tag: 'h1', classes: ['main-title'], attributes: { id: 'top' } }
    },
    {
        // '~=' matches if the word 'Quote' appears anywhere in the style name
        selector: { attributes: { style: { value: 'Quote', operator: '~=' } } },
        output: { tag: 'blockquote', fresh: true }
    }
]

fresh: true prevents the generator from merging adjacent nodes of the same tag into one block. Without it, consecutive paragraphs mapped to blockquote would be combined into a single element.

ChunkingConfig Reference

ChunkingConfig is a discriminated union; available options depend on the strategy field. Default strategy is 'document-structure'.

stripWhitespace boolean true Trim leading/trailing whitespace from each chunk.
includeMetadata boolean true Include page/slide/heading metadata in each chunk object.
addStartIndex boolean false Include startIndex and endIndex character offsets at the root of the chunk object.

strategy: 'document-structure'

Option Type Default Description
