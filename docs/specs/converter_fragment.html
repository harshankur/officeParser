OfficeConverter Specification

The OfficeConverter is the highest-level API in the library. It provides a simple, "one-step" method to transform documents from one format to another without manually handling the intermediate AST.

OfficeConverter.convert()

Converts a source document to a destination format in a single asynchronous call.

import { OfficeConverter } from 'officeparser';

const { value, messages } = await OfficeConverter.convert('report.docx', 'pdf', {
    generatorConfig: {
        includeFormatting: true,
        pdfConfig: { format: 'A4' }
    }
});

Arguments

Argument Type Description
source string | Buffer | ArrayBuffer The source file path or binary data.
destination UniversalGeneratorFormat Target format: 'html', 'md', 'pdf', 'csv', 'text', 'rtf', 'chunks'.
config OfficeConverterConfig Optional combined configuration for parser and generator.

OfficeConverterConfig

Combines settings for both the parsing phase and the generation phase.

Property Type Default Description
parseConfig OfficeParserConfig {} Settings for how the source file is parsed.
generatorConfig GeneratorConfig {} Settings for how the output is generated.
onWarning function undefined Global callback for issues in either phase. Overrides phase-specific onWarning callbacks.
[!TIP] Use OfficeConverter for simple CLI tools or scripts where you only need a single output. For complex pipelines (e.g., generating both HTML and Markdown from the same file), use OfficeParser followed by OfficeGenerator to avoid redundant parsing.
