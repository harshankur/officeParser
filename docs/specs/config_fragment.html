officeParser Configuration Specification

This document details the configuration options available for the officeParser library. These options directly influence how the document structure is interpreted and what data is extracted into the Final AST.

OfficeParserConfig

The OfficeParserConfig object is used as the second argument to parseOffice. All properties are optional.

Property Type Default Description (In Action)
outputErrorToConsole boolean false Diagnostic Mode: When enabled, the library will dump internal trace logs and raw parsing errors to the browser/node console. Essential for debugging malformed files.
newlineDelimiter string \n Text Flattening: Determines the string inserted between structural blocks (paragraphs, cells, slides) when calculating the pre-flattened .text property. Change to <br> for HTML-ready output.
ignoreNotes boolean false Note Filtering: Specifically targets PowerPoint (PPTX/ODP) speaker notes. If true, notes are entirely omitted from both the structure and text output.
putNotesAtLast boolean false Note Reordering: By default, notes are interleaved after each slide. If true, all notes are collected and appended as a final section at the end of the content tree.
extractAttachments boolean false Resource Extraction: Enables the collection of images, charts, and embedded objects. Populates the root attachments array and ensures image nodes have attachmentName metadata.
includeRawContent boolean false Source Inspection: Adds a rawContent property to every node containing its original XML or RTF source. Warning: Significantly increases AST memory footprint.
serializeRawContent boolean true XML Cleanliness: When includeRawContent is true, re-serializes raw XML to clean strings. If false, extracts original raw substring.
preserveXmlWhitespace boolean false Formatting Fidelity: When serializeRawContent is true, preserves original XML whitespace and line endings during serialization.
ocr boolean false Vision Synthesis: Enables optical character recognition for images and non-selectable PDF layers. Requires Tesseract.js.
ocrLanguage string eng Deprecated: Use ocrConfig.language. Specifies which Tesseract data files to load.
pdfWorkerSrc string (CDN) PDF Engine: Path to the pdf.worker.js. Must be accessible by the browser.
ocrConfig object {} OCR Scheduler: Fine-grained settings for the intelligent worker pool.
ocrConfig.language string eng Language Training: Specifies Tesseract languages (e.g., 'eng+fra').
ocrConfig.autoTerminateTimeout number 10000 Lifecycle Safety: Inactivity timeout in ms before workers are killed. Set to `0` to disable.
ocrConfig.workerPath / corePath / langPath string undefined Offline Environments: Paths to Tesseract scripts and data files for air-gapped or customized installations.
includeBreakNodes boolean false Line/Page Break Parsing: Specifically targets Word documents (DOCX). When set to true, officeParser will extract physical breaks into the AST as break nodes. This includes standard line breaks (textWrapping), hard carriage returns (carriageReturn), page breaks (page), column breaks (column), and soft layout markers (lastRenderedPage).

OCR Scheduler & Resource Management (Smart Pool)

Using OCR initializes a persistent **Smart Worker Pool** to improve performance for subsequent requests.

  • Dynamic Affinity: Workers persist with their language affinity.
  • Smart Re-initialization: If a new language is requested at capacity, the Least Recently Used (LRU) idle worker is re-initialized instead of a full pool reset.
  • Automatic Cleanup: Workers are killed after the configured timeout of inactivity.

OfficeParser.terminateOcr()

A static method that immediately kills all active OCR workers. Call this in short-lived scripts (CLI) to bypass the inactivity timer and exit the terminal immediately.

await OfficeParser.terminateOcr();

Common Scenarios

1. Search Engine Indexing (High Speed)

const config = {
    ignoreNotes: true,
    extractAttachments: false, // Don't need images for indexing
    newlineDelimiter: ' '      // Simple space for search ranking
};

2. Document Reconstruction (Rich Data)

const config = {
    extractAttachments: true,
    includeRawContent: true,   // For deep attribute analysis
    putNotesAtLast: false      // Keep slides and notes together
};
[!IMPORTANT] RTF Notes: putNotesAtLast is currently forced to true for RTF files due to the nature of the format's group stacking for footnotes/endnotes.
