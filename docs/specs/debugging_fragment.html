officeParser Debugging Tips

Tips and common solutions for working with officeParser.

1. Enable Verbose Errors

If you encounter an issue during parsing, set outputErrorToConsole to true. This will log internal stack traces.

const ast = await officeParser.parseOffice(file, { outputErrorToConsole: true });

2. Accessing Raw Source

Enable includeRawContent to attach original XML/RTF markup to each node. Useful for investigating missing formatting or internal tags.

3. PDF Parsing in Browser

In the browser, PDF parsing requires a background worker script. Default is a CDN, but you can host it locally: 

const config = {
    pdfWorkerSrc: '/assets/workers/pdf.worker.min.mjs'
};

4. OCR Issues

  • Multiple Languages: Provide comma-separated languages (e.g., eng+fra).
  • Image Quality: OCR accuracy depends on resolution and clarity.
  • Connectivity: Tesseract.js may need to download data files.

5. Large Spreadsheet Memory

XLSX files can be massive. Use ast.toText() if you only need plain text to save memory, or parse on the server.
