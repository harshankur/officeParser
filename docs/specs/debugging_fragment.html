Troubleshooting & Debugging

Solutions for common integration challenges and performance optimizations in officeParser v7.0.0+.

Pro Tip
Use the onWarning callback to capture internal diagnostic information and trace logs during the parsing process. 
const config = {
    onWarning: (issue) => console.warn(`[OfficeParser] ${issue.message}`, issue.details)
};

1. Node.js Process Hanging (OCR)

In v6.1.0, if you use ocr: true, the library maintains a persistent worker pool to make subsequent OCR tasks much faster. However, this background pool will keep your Node.js process alive for 10 seconds of inactivity by default.

Solutions:

  • For CLI/Short-lived scripts: Call await terminateOcr() manually after your parseOffice call.
  • Change Timeout: Set a shorter idle period via ocrConfig.autoTerminateTimeout.
  • Disable Pooled Persistence: Set autoTerminateTimeout: 0 to kill workers immediately (similar to legacy behavior).
import { parseOffice, terminateOcr } from 'officeparser';

const ast = await parseOffice('file.png', { ocr: true });
// Process finishes...
await terminateOcr(); // Snappy exit

2. PDF Rendering Issues (Browser)

If PDF parsing fails in the browser with "Worker not found" or "Version mismatch" errors:

  • Worker Source: Ensure pdfWorkerSrc points to a valid URL. By default, it uses a CDN pointing to version 5.6.205.
  • Version Matching: If you host the worker locally, ensure it matches the pdfjs-dist version 5.6.205 used by the library.

3. OCR Accuracy & Performance

If OCR results are missing or incorrect:

  • Language Selection: Ensure you've provided the correct language code. You can combine multiple languages with a + sign (e.g., eng+deu).
  • Air-gapped Environments: In restricted environments, tesseract.js cannot download language files. You must provide local paths via ocrConfig.workerPath, ocrConfig.corePath, and ocrConfig.langPath.
  • Memory: Parallel OCR on many images can be intensive. officeParser limits the pool to 4 concurrent workers by default to prevent crashes.

4. Memory Management (Giant Files)

When parsing extremely large Excel (.xlsx) or PowerPoint (.pptx) files:

  • AST vs. Text: The structured AST consumes significantly more memory than plain text. If you don't need the hierarchy, use ast.toText() and allow the AST to be garbage-collected immediately.
  • Ignore Notes: For complex PowerPoints, setting ignoreNotes: true can reduce memory usage by skipping entire XML trees.

5. Metadata & Custom Properties

If metadata.customProperties is empty:

  • Custom property extraction is currently limited to OOXML (DOCX, XLSX, PPTX), ODF (ODT, ODS, ODP), and PDF.
  • Legacy formats like RTF do not support custom property extraction.

6. Bundle Selection

Ensure you are importing the correct build for your environment:

  • Node.js (CJS/ESM): Standard import or require works automatically via package.json exports.
  • Modern Bundlers (Vite/Webpack): Use the .mjs bundle for best treeshaking.
  • Legacy Browser Scripts: Use dist/officeparser.browser.iife.js for a global officeParser namespace.

7. Text-Based Buffer Detection Failure (IMPROPER_BUFFERS)

When passing a Buffer or ArrayBuffer for md, html, or csv files, the parser cannot auto-detect the format — these formats have no magic bytes. You must provide a fileType hint: 

// Without fileType → throws IMPROPER_BUFFERS
await officeParser.parseOffice(markdownBuffer);

// Correct:
await officeParser.parseOffice(markdownBuffer, { fileType: 'md' });
await officeParser.parseOffice(htmlBuffer,     { fileType: 'html' });
await officeParser.parseOffice(csvBuffer,      { fileType: 'csv' });

The fileType option is authoritative — it bypasses magic-bytes detection even for binary formats.

8. PDF Generation Fails (Node.js)

PDF output (format: 'pdf') uses Puppeteer, which is an optional peer dependency not included by default. If generation throws a dependency error, install it:

npm install puppeteer

In CI/CD environments where Puppeteer cannot download Chromium automatically, pass a custom executable path via pdfConfig.launchOptions: 

await OfficeGenerator.generate(ast, 'pdf', {
    pdfConfig: {
        launchOptions: {
            executablePath: '/usr/bin/chromium-browser',
            args: ['--no-sandbox', '--disable-setuid-sandbox']
        }
    }
});

Quick Reference

md/html/csv buffer not detected Add fileType: 'md' (or 'html', 'csv') to config
