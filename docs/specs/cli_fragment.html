CLI Options Reference

This document details the command line interface options available when running officeparser via npx or globally installed scripts.

Basic Usage

npx officeparser <file> [options]

Options can be passed in multiple formats: --key=value, --key value, as bare flags (e.g. --ocr which defaults to true), or negation flags (e.g. --no-ocr). Most options map directly to the OfficeParserConfig and GeneratorConfig properties, and you can access nested properties using JSON dot-notation (e.g. --htmlConfig.containerWidth=900px).

Available Flags

Flag Values Default Description
--to json | text | md | html | csv | rtf | pdf | epub | chunks json The desired output format for the parsed document.
--output path — Write the generated output to a specified file instead of stdout.
--fileType docx | xlsx | pptx | odt | odp | ods | pdf | rtf | csv | md | html | epub — Explicitly override input file type detection (useful when input file lacks extension).
--ocr boolean false Enable Optical Character Recognition (OCR) for embedded images.
--extractAttachments boolean false Extract embedded images and charts as Base64 data URIs.
--ignoreNotes boolean false Ignore footnotes, endnotes, and speaker notes.
--ignoreComments boolean false Ignore inline comments and threaded replies.
--ignoreHeadersAndFooters boolean false Ignore page headers and footers.
--ignoreSlideMasters boolean false Ignore slide master layouts and static template elements (PPTX/ODP).
--ignoreInternalLinks boolean false Skip internal anchors and bookmark links.
--newlineDelimiter string \n Delimiter string to use between lines/blocks in plaintext outputs.
--csvDelimiter string , Custom delimiter character for CSV files.
--includeRawContent boolean false Include the raw XML/RTF format strings within parsed nodes.
--serializeRawContent boolean true Include raw XML metadata directly in the AST properties.
--preserveXmlWhitespace boolean false Keep raw formatting spaces in XML parsing.
--includeBreakNodes boolean false Include explicit page/column break nodes (DOCX only).
--verbose boolean false Show full error stack traces and warning logs on failures.
--includeFormatting boolean true Include formatting style map matching for generators.
--renderMetadata boolean false Render metadata block as visible content in the generated output.
--htmlConfig.containerWidth string | number auto HTML output container width (e.g. 900px, 100%, auto).
--format json | text | md | html | csv | rtf | pdf | epub | chunks json Deprecated. Use --to instead.
--toText boolean false Deprecated. Use --to=text instead, which keeps footnote text and image placeholders by default (both switchable); this flag drops them unconditionally.
--ocrLanguage string eng Deprecated. Use --ocrConfig.language instead.
--putNotesAtLast boolean false Deprecated and ignored. Notes are attached structurally to their nodes.
--outputErrorToConsole boolean false Deprecated. Use --verbose instead.

Examples

Generate HTML Preview

npx officeparser report.docx --to html --output preview.html --htmlConfig.containerWidth=800px

Extract RAG Chunks with OCR

npx officeparser presentation.pptx --to chunks --ocr --ocrConfig.language=eng --output chunks.json

Plain Text Extraction

npx officeparser data.pdf --to text --ignoreNotes

Convert DOCX to EPUB

npx officeparser book.docx --extractAttachments --to epub --output book.epub
[!IMPORTANT] --extractAttachments is required when converting to EPUB if the document has images — without it, no image bytes are available for the EPUB generator to package.
